## 一、系统概述

### 1.1 项目背景

药品上市是医药企业最核心、风险最高的决策之一。传统方式依赖：
- 医生调研（耗时3-6个月，样本有限）
- 历史类比分析（缺乏可比性）
- 专家经验判断（主观偏差大）

**核心痛点：**
- 一个新药从临床到上市，市场预测偏差可达 30%-50%
- 竞品突然上市、医保政策变化等事件难以预判
- 不同定价策略、推广力度的效果无法提前验证

### 1.2 解决方案

PharmaSim 通过构建**多智能体仿真系统**，模拟药品上市后各利益相关方的行为反应：
- **医生 Agent**：模拟处方决策行为
- **患者 Agent**：模拟就医、用药依从性行为
- **医保 Agent**：模拟医保谈判和报销政策影响
- **渠道 Agent**：模拟药房、电商平台的进货和推荐行为
- **竞品 Agent**：模拟竞品企业的市场应对策略

### 1.3 与 Aaru 的差异定位

| 维度 | Aaru | PharmaSim |
|------|------|-----------|
| 行业 | 通用（商业/政治/政府） | 深耕医药医疗 |
| 数据源 | 交易数据、选票数据 | 处方数据、医保数据、临床数据 |
| Agent 类型 | 通用消费者 | 医生/患者/医保/渠道 |
| 输出 | 通用市场洞察 | 药品上市KPI预测（处方量、渗透率、收入） |
| 合规要求 | 美国隐私法 | 中国《药品管理法》《个人信息保护法》 |

├── 处方选择（医生）
├── 用药决策（患者）
├── 报销政策（医保）
└── 渠道策略（药房/电商）

## 五、关键技术挑战与解决方案

### 挑战1：Agent 行为一致性

**问题：** LLM 输出具有随机性，同一个 Agent 在不同时间可能做出矛盾决策。

**解决方案：**
- 为每个 Agent 维护**记忆系统**（向量数据库存储历史决策）
- 决策时注入 Agent 的历史行为作为约束
- 设置行为参数的确定性边界（温度降低 + 规则约束）

### 挑战2：规模化性能

**问题：** 模拟 10,000+ Agent 的 LLM 调用成本和时间。

**解决方案：**
- Agent 分层：核心 Agent（高精度LLM）+ 普通 Agent（规则+小模型）
- 批量推理优化
- 增量模拟：只在关键时间点触发全量模拟
- 缓存相似 Agent 的决策结果

### 挑战3：数据获取

**问题：** 处方数据、医保数据敏感且难以获取。

**解决方案：**
- Phase 1：使用公开数据（PDB、药智网、CDE）
- Phase 2：与药企合作，用脱敏数据训练
- Phase 3：建立数据合作网络（医院、医保局、第三方数据商）

### 挑战4：结果可信度

**问题：** 客户为什么相信 AI 模拟的结果？

**解决方案：**
- 回测验证：用已上市药品的真实数据背书
- 置信区间：每个预测都给出不确定性范围
- 可解释性：每个决策都能追溯到具体 Agent 的推理过程
- 渐进式信任：先做辅助参考，逐步替代传统调研

## 六、核心机制：Aaru式多Agent社交交互 (v3.0)

> 仿照 [Aaru](https://aaru.com) 的多智能体仿真方法论，PharmaSim 实现了用户Agent间的社交网络交互与决策涌现。以下五个核心机制构成了系统的差异化竞争力。

### 6.1 丰富人设 — 每个Agent有完整的心理、行为、人口学画像

每个患者Agent不只是一个数字，而是拥有**多维度个体画像**的虚拟人：

```
PatientAgent:
├── 人口学属性
│   ├── 年龄、性别、地区
│   ├── 经济水平 (高/中/低收入)
│   └── 医保类型 (城镇职工/居民/新农合/商保/自费)
├── 疾病属性
│   ├── 疾病类型与分期 (早期/中期/晚期)
│   └── 依从性评分 (0-1)
├── 心理属性
│   ├── 品牌偏好 (原研药/仿制药/无偏好)
│   ├── 信息来源偏好 (医生/病友/网络/家人)
│   └── 风险厌恶度 (隐含在经济水平中)
└── 行为属性
    ├── 决策风格 (冲动/理性/从众)
    └── 社交活跃度 (病友社区参与度)
```

**设计意义：** 人设的丰富度决定了模拟的真实度。Aaru 验证了"如果你能描述一个人，就能模拟一个人"的理念。PharmaSim 将这一理念应用到医疗场景——每个Agent的决策都受到其独特画像的调制，而非所有人做出相同反应。

### 6.2 社交网络 — Agent之间有连接关系（小世界网络）

Agent 不是孤立决策的，它们被组织在一个**小世界网络 (Watts-Strogatz)** 中：

```
社交网络拓扑:
├── 患者-患者连接 (小世界网络)
│   ├── 同地区邻居: 信任度 0.6-0.9
│   ├── 同病友社区: 信任度 0.5-0.9
│   └── 随机弱连接: 信任度 0.3-0.7
├── 医生-患者连接 (KOL模型)
│   ├── 医生作为枢纽节点 (Hub)
│   └── 每个医生影响 5-15 个患者
├── 专家-患者连接 (弱连接)
│   └── 10%概率连接专家作为信息源
└── 网络特征
    ├── 节点数: 1085
    ├── 边数: 12000+
    ├── 平均度数: 11.2
    ├── 枢纽节点: 104个 (KOL)
    └── 社区数: 3个 (病友社区)
```

**设计意义：** 小世界网络是真实社交关系的经典模型——每个人有紧密的本地连接，也有少量远程"弱连接"。这确保了信息既能局部传播，也能全局扩散。

### 6.3 意见传播 — 通过社交网络影响邻居的决策

信息和意见在社交网络中像"病毒"一样传播，影响力随距离衰减：

```
影响强度计算:
influence = edge_weight × confidence_gap × opinion_similarity × time_decay × hub_bonus

其中:
├── edge_weight: 社交连接权重 (信任度 × 交互频率 × 关系类型系数)
├── confidence_gap: 信心差距 (高信心者影响低信心者)
├── opinion_similarity: 同质偏好 (相似意见更容易被接受)
├── time_decay: 时间衰减 (e^(-0.1×轮次))
└── hub_bonus: 枢纽节点加成 (KOL影响力 ×1.5)
```

此外，系统模拟**外部事件冲击**：
- 第2轮：媒体报道不良反应 → 30%患者意愿下降
- 第3轮：KOL公开推荐 → 20%患者意愿回升
- 第5轮：病友社区正面反馈 → 40%患者受影响
- 第7轮：医保谈判结果 → 60%患者最终确认

### 6.4 权重匹配 — 影响力=专业度×置信度×社交中心度

不是每个Agent的意见都同等重要。影响力由三个维度决定：

```
Agent 影响力 = f(专业度, 置信度, 社交中心度)

├── 专业度 (Expertise)
│   ├── 药物学专家: 对药理学维度权重高
│   ├── 临床专家: 对疗效/安全性维度权重高
│   ├── 医保专家: 对准入/报销维度权重高
│   └── 患者: 对价格/可及性维度权重高
├── 置信度 (Confidence)
│   ├── 受信息充分度影响 (医生推荐 → 高置信)
│   ├── 受专业水平影响 (权威专家 → 高置信)
│   └── 每轮交互后置信度微增 (+0.02)
└── 社交中心度 (Centrality)
    ├── 枢纽节点 (KOL): 度数前5%，影响力 ×1.5
    ├── 活跃社区成员: 病友社区核心人物
    └── 普通节点: 基础影响力
```

**患者维度关注度权重：**
| 维度 | 低收入患者 | 高收入患者 | 通用 |
|------|-----------|-----------|------|
| 医保评估 | 0.95 | 0.9 | 0.9 |
| 定价评估 | 0.95 | 0.4 | 0.8 |
| 临床评估 | 0.7 | 0.7 | 0.7 |

### 6.5 多轮协商 — 决策在多轮交互中涌现

最终决策不是一次性计算出来的，而是在**多轮协商**中逐步涌现：

```
协商流程:
Round 0: 意见初始化 (基于Agent个体画像 + 药品属性 + 专家评估)
    ↓
Round 1: 社交信息交换 → 部分Agent调整意见
    ↓ [外部冲击: 媒体负面报道]
Round 2: 意见进一步趋同或分化
    ↓ [外部冲击: KOL推荐]
Round 3: 群体意见开始收敛
    ↓
Round 4-6: 稳定期，微调
    ↓ [外部冲击: 病友反馈 + 医保结果]
Round 7-8: 最终收敛 → 涌现出群体决策

收敛条件: |本轮均值 - 上轮均值| < 0.01
```

**涌现行为示例：**
- 初始意愿 0.785 → 媒体冲击降至 0.766 → KOL推荐回升至 0.776 → 收敛
- 积极采纳者: 750人 (75%) | 谨慎观望者: 243人 (24.3%) | 中立: 7人 (0.7%)
- 群体意见从个体差异中涌现，而非简单平均
